# Техническое задание на разработку платформы NeuroAdmin

## Общая информация

**Название проекта:** NeuroAdmin  
**Тип проекта:** AI-Powered SMM Platform  
**Версия ТЗ:** 1.0  
**Дата создания:** 13 декабря 2025  
**Статус:** В разработке

---

## 1. Общее описание проекта

### 1.1 Назначение системы

NeuroAdmin — это комплексная SMM-платформа, использующая технологии искусственного интеллекта для автоматизации задач управления социальными сетями. Платформа предоставляет единый интерфейс для работы с различными социальными сетями и выполнения автоматизированных задач по управлению аудиторией и контентом.

### 1.2 Целевая аудитория

**Основные пользователи:**

- SMM-менеджеры
- Владельцы и администраторы сообществ в социальных сетях
- Маркетологи, работающие с социальными медиа
- Агентства, предоставляющие SMM-услуги

**Требования к пользователям:**

- Наличие аккаунтов в социальных сетях ВКонтакте и/или Одноклассники
- Права администратора или модератора в сообществах (для некоторых функций)
- Базовые навыки работы с веб-интерфейсами

### 1.3 Цели и задачи системы

**Основные цели:**

1. Автоматизация рутинных задач управления социальными сетями
2. Улучшение качества аудитории сообществ за счет очистки от ботов
3. Централизованное управление несколькими социальными сетями
4. Предоставление аналитики и отчетности по работе с аудиторией

**Задачи системы:**

- Подключение и управление аккаунтами социальных сетей
- Создание и выполнение различных типов задач для управления сообществами
- Анализ аудитории с использованием AI-технологий
- Предоставление детальной статистики и отчетов
- Экспорт данных для дальнейшего анализа

## 2. Функциональные требования

### 2.1 Модуль аутентификации и управления аккаунтом

#### 2.1.1 Регистрация пользователя

**Требования:**

- Регистрация через форму с полями:
  - Email (обязательное, валидация формата)
  - Пароль (обязательное, минимум 8 символов)
  - Подтверждение пароля (обязательное, должно совпадать)
  - Имя (обязательное)
  - Фамилия (обязательное)
- Валидация всех полей на клиенте и сервере
- Проверка уникальности email
- Автоматическая авторизация после успешной регистрации
- Генерация JWT токена при регистрации

**Обработка ошибок:**

- Email уже зарегистрирован → сообщение "Email уже зарегистрирован"
- Пароль не соответствует требованиям → детальное сообщение о требованиях
- Пароли не совпадают → сообщение "Пароли не совпадают"

#### 2.1.2 Вход в систему

**Требования:**

- Вход через форму с полями:
  - Email
  - Пароль
- Валидация учетных данных
- Генерация JWT токена при успешном входе
- Сохранение токена в браузере пользователя
- Автоматическое перенаправление на Dashboard после входа

**Обработка ошибок:**

- Неверный email или пароль → сообщение "Неверный email или пароль" (без указания, что именно неверно)
- Валидация формы → отображение ошибок для соответствующих полей

#### 2.1.3 Выход из системы

**Требования:**

- Кнопка выхода в меню пользователя
- Удаление JWT токена из браузера
- Перенаправление на страницу входа

#### 2.1.4 Восстановление пароля

**Требования:**

- Форма запроса сброса пароля (поле Email)
- Генерация токена сброса пароля
- Отправка email с ссылкой для сброса (будущая версия)
- Форма сброса пароля с полями:
  - Новый пароль
  - Подтверждение нового пароля
- Валидация токена сброса
- Обновление пароля в базе данных

**Обработка ошибок:**

- Email не найден → сообщение "Если аккаунт существует, письмо будет отправлено" (безопасность)
- Токен истек → сообщение "Ссылка недействительна или истекла"

#### 2.1.5 Управление профилем

**Требования:**

- Просмотр информации профиля:
  - Email (только для просмотра)
  - Имя
  - Фамилия
  - Дата регистрации
- Редактирование профиля (Имя, Фамилия)
- Смена пароля:
  - Текущий пароль
  - Новый пароль
  - Подтверждение нового пароля
- Валидация всех изменений

### 2.2 Модуль подключения социальных сетей

#### 2.2.1 Подключение аккаунта ВКонтакте

**Требования:**

- Раздел на Dashboard для подключения социальных сетей
- Карточка "ВКонтакте" со статусом "Не подключено"
- Кнопка "Подключить" на карточке
- Модальное окно с информацией о запрашиваемых правах доступа
- OAuth авторизация через VK API
- Получение access token и refresh token
- Получение информации о пользователе из VK API
- Сохранение токенов и информации об аккаунте в БД
- Обновление статуса карточки на "Подключено"
- Отображение подключенного аккаунта в списке

**Обработка ошибок:**

- Отмена авторизации → возврат на Dashboard без ошибок
- Ошибка OAuth → сообщение "Не удалось подключить аккаунт. Попробуйте еще раз"
- Недостаточно прав → сообщение "Необходимо предоставить все запрашиваемые права доступа"

#### 2.2.2 Подключение аккаунта Одноклассники

**Требования:**

- Аналогично подключению VK
- OAuth авторизация через OK API
- Сохранение токенов и информации

#### 2.2.3 Просмотр списка подключенных аккаунтов

**Требования:**

- Раздел "Подключенные аккаунты" на Dashboard
- Отображение всех подключенных аккаунтов:
  - Иконка социальной сети
  - Имя пользователя / название сообщества
  - Аватар/логотип
  - Дата подключения
  - Статус подключения (активен/неактивен)
- Группировка по социальным сетям (VK, Одноклассники)
- Пустое состояние с подсказкой "Подключите социальную сеть, чтобы начать работу"

#### 2.2.4 Просмотр деталей подключенного аккаунта

**Требования:**

- Модальное окно или страница с деталями:
  - Иконка социальной сети
  - Имя пользователя / название сообщества
  - Аватар/логотип
  - Дата подключения
  - Статус подключения
  - Количество созданных задач для этого аккаунта
  - Список сообществ/групп (для VK)
  - Информация о последней активности

#### 2.2.5 Отключение аккаунта социальной сети

**Требования:**

- Кнопка "Отключить" на карточке аккаунта
- Диалог подтверждения с предупреждением:
  - "Вы уверены, что хотите отключить этот аккаунт?"
  - "Все невыполненные задачи для этого аккаунта будут отменены"
- Автоматическая отмена всех активных задач для этого аккаунта
- Удаление токенов доступа из БД
- Удаление информации об аккаунте
- Обновление статуса карточки социальной сети

#### 2.2.6 Обновление токена доступа (автоматическое)

**Требования:**

- Автоматическое определение истечения токена
- Использование refresh token для получения нового access token
- Обновление токенов в БД
- Прозрачно для пользователя
- При истечении refresh token → пометка аккаунта как неактивного, уведомление пользователя

### 2.3 Модуль системы задач

#### 2.3.1 Просмотр списка задач

**Требования:**

- Страница "Задачи" с списком всех задач пользователя
- Карточка задачи с информацией:
  - Тип задачи (иконка + название)
  - Социальная сеть и аккаунт
  - Статус (Pending, InProgress, Completed, Failed, Cancelled)
  - Дата создания
  - Дата завершения (если завершена)
  - Прогресс выполнения (для активных задач)
- Сортировка по дате создания (новые сверху)
- Пагинация или бесконечная прокрутка
- Пустое состояние: "У вас пока нет задач" с кнопкой "Создать задачу"

#### 2.3.2 Фильтрация задач

**Требования:**

- Панель фильтров над списком задач
- Фильтры:
  - По типу задачи (Очистка от ботов, Статистика сообщества, и т.д.)
  - По социальной сети (VK, Одноклассники)
  - По статусу (Pending, InProgress, Completed, Failed, Cancelled)
  - По дате создания (сегодня, неделя, месяц, все время)
  - По конкретному аккаунту
- Применение фильтров автоматически или по кнопке
- Отображение количества найденных задач
- Кнопка "Сбросить фильтры"

#### 2.3.3 Поиск задач

**Требования:**

- Поле поиска над списком задач
- Поиск по:
  - Названию задачи
  - Типу задачи
  - Названию аккаунта/сообщества
- Поиск в реальном времени (при вводе)
- Сообщение "По запросу '{ключевое слово}' ничего не найдено" при отсутствии результатов

#### 2.3.4 Просмотр деталей задачи

**Требования:**

- Страница деталей задачи с информацией:
  - Тип задачи
  - Социальная сеть и аккаунт
  - Статус задачи
  - Дата создания
  - Дата начала выполнения
  - Дата завершения (если завершена)
  - Параметры задачи (что было задано при создании)
  - Прогресс выполнения (для активных задач)
  - Информация об ошибках (если статус Failed)
- Кнопка "Просмотреть результаты" для завершенных задач
- Кнопка "Отменить задачу" для активных задач
- Кнопка "Удалить задачу" для завершенных/отмененных задач

#### 2.3.5 Отмена задачи

**Требования:**

- Кнопка "Отменить задачу" на странице деталей активной задачи
- Диалог подтверждения:
  - "Вы уверены, что хотите отменить эту задачу?"
  - "Прогресс выполнения будет потерян"
- Остановка обработки задачи
- Обновление статуса на "Cancelled"
- Уведомление "Задача отменена"

#### 2.3.6 Удаление задачи

**Требования:**

- Кнопка "Удалить задачу" для завершенных/отмененных задач
- Диалог подтверждения:
  - "Вы уверены, что хотите удалить эту задачу?"
  - "Действие необратимо. Все результаты будут удалены"
- Удаление задачи из БД
- Удаление связанных результатов
- Перенаправление на страницу списка задач

### 2.4 Модуль очистки от ботов

#### 2.4.1 Создание задачи "Очистка от ботов"

**Требования:**

- Форма создания задачи в системе задач
- Выбор типа задачи: "Очистка от ботов"
- Поля формы:
  - Выбор социальной сети (VK или Одноклассники) - выпадающий список
  - Выбор аккаунта/сообщества - выпадающий список подключенных аккаунтов выбранной соц сети
  - Выбор сообщества/группы (если несколько доступно) - выпадающий список
  - Дополнительные параметры (опционально):
    - Порог Bot Score (по умолчанию 0.7)
    - Выбор стратегий определения ботов (чекбоксы):
      - Анализ активности
      - Анализ профиля
      - Анализ друзей/подписчиков
      - Анализ контента
    - Дополнительные фильтры
- Валидация всех обязательных полей
- Создание задачи со статусом "Pending"
- Публикация задачи в очередь RabbitMQ для асинхронной обработки
- Перенаправление на страницу деталей созданной задачи

**Обработка ошибок:**

- Нет подключенных аккаунтов → предупреждение "У вас нет подключенных аккаунтов" с предложением подключить
- Нет доступных сообществ → предупреждение "У выбранного аккаунта нет доступных сообществ"
- Ошибка валидации → отображение ошибок для соответствующих полей
- Ошибка при создании → сообщение "Не удалось создать задачу. Попробуйте еще раз"

#### 2.4.2 Мониторинг выполнения задачи очистки от ботов

**Требования:**

- Страница деталей задачи с информацией:
  - Статус: "Выполняется" (InProgress)
  - Прогресс-бар с процентами выполнения
  - Количество проверенных пользователей / общее количество
  - Количество найденных ботов (если уже есть результаты)
  - Время начала выполнения
  - Примерное время до завершения (если можно рассчитать)
- Автоматическое обновление информации о прогрессе (polling каждые 5-10 секунд или WebSocket)
- Обновление статуса на "Completed" при завершении
- Уведомление "Задача завершена"
- Кнопка перехода к просмотру результатов

**Обработка ошибок:**

- Задача провалена → статус "Failed" с информацией об ошибке
- Задача отменена → статус "Cancelled", остановка прогресса

#### 2.4.3 Просмотр результатов очистки от ботов

**Требования:**

- Страница результатов с разделами:

  **Общая статистика:**

  - Общее количество проверенных пользователей
  - Количество найденных ботов
  - Количество реальных пользователей
  - Процент ботов
  - Время выполнения задачи

  **Визуализация:**

  - Круговая диаграмма (боты / реальные пользователи)
  - График распределения Bot Scores
  - Временная шкала выполнения задачи

  **Таблица найденных ботов:**

  - Колонки: ID, Имя, Аватар, Bot Score, Социальная сеть, Действия
  - Сортировка по Bot Score (по умолчанию)
  - Пагинация (50 записей на страницу)

- Пустое состояние: "Ботов не обнаружено. Аудитория чистая!" (если ботов не найдено)

#### 2.4.4 Фильтрация и поиск в результатах

**Требования:**

- Панель фильтров и поиска над таблицей ботов
- Поиск по имени/ID
- Фильтр по Bot Score (диапазон, например, 0.7-0.9, 0.9-1.0)
- Фильтр по статусу (все боты / только боты / только подозрительные)
- Сортировка:
  - По Bot Score (по возрастанию/убыванию)
  - По имени
  - По дате обнаружения
- Отображение количества найденных результатов
- Кнопка "Сбросить фильтры"

#### 2.4.5 Просмотр деталей конкретного бота

**Требования:**

- Модальное окно с детальной информацией о пользователе:

  **Основная информация:**

  - Фото профиля (большое)
  - Имя пользователя
  - ID пользователя
  - Ссылка на профиль в социальной сети
  - Bot Score (общий)

  **Детали анализа:**

  - Breakdown Bot Score по стратегиям:
    - Анализ активности: X баллов
    - Анализ профиля: X баллов
    - Анализ друзей/подписчиков: X баллов
    - Анализ контента: X баллов
  - Описание причин, почему пользователь определен как бот

  **Информация о пользователе:**

  - Количество друзей/подписчиков
  - Количество постов/публикаций
  - Дата регистрации аккаунта
  - Последняя активность
  - Основная информация из профиля

  **История активности** (если доступна)

- Кнопка копирования ссылки на профиль
- Кнопка перехода к профилю в социальной сети (открыть в новой вкладке)

#### 2.4.6 Экспорт результатов очистки от ботов

**Требования:**

- Кнопка "Экспорт результатов"
- Меню выбора формата экспорта:
  - CSV
  - Excel (XLSX)
  - PDF
  - JSON
- Выбор данных для экспорта:
  - Все результаты
  - Только боты (фильтр по Bot Score >= порога)
  - Текущий фильтр (если применен)
- Генерация файла с результатами:
  - CSV/Excel: таблица с колонками (ID, Имя, Bot Score, Детали анализа)
  - PDF: отчет с графиками и таблицей
  - JSON: структурированные данные
- Автоматическое скачивание файла или предоставление ссылки

**Обработка ошибок:**

- Большой объем данных → показ прогресса генерации файла
- Ошибка при экспорте → сообщение "Не удалось экспортировать результаты"

#### 2.4.7 Повторный анализ (перезапуск задачи)

**Требования:**

- Кнопка "Повторить анализ" или "Создать аналогичную задачу" на странице результатов
- Форма создания задачи с предзаполненными параметрами из предыдущей задачи:
  - Социальная сеть
  - Аккаунт/сообщество
  - Порог Bot Score
  - Выбранные стратегии
- Возможность изменения параметров
- Создание новой задачи

### 2.5 Будущие типы задач (планируются)

#### 2.5.1 Получение статистики сообщества

**Требования (будущая версия):**

- Форма создания задачи:
  - Выбор социальной сети
  - Выбор сообщества
  - Выбор периода (неделя, месяц, квартал, год, кастомный диапазон)
  - Выбор типа статистики (подписчики, просмотры, охваты, взаимодействия)
- Выполнение задачи (синхронно или асинхронно)
- Отображение результатов: графики и таблицы со статистикой

#### 2.5.2 Создание поста в сообществе

**Требования (будущая версия):**

- Форма создания задачи:
  - Выбор социальной сети
  - Выбор сообщества
  - Текст поста
  - Загрузка медиа (фото, видео)
  - Настройки публикации:
    - Опубликовать сейчас
    - Запланировать публикацию (дата и время)
    - Прикрепить геолокацию
    - Настройки комментариев
- Выполнение задачи:
  - Если "Опубликовать сейчас" → публикация немедленно
  - Если "Запланировать" → публикация в указанное время
- Отображение статуса публикации и ссылки на опубликованный пост

#### 2.5.3 Управление комментариями

**Требования (будущая версия):**

- Форма создания задачи:

## 6. Требования к обработке ошибок

### 6.1 Обработка ошибок API социальных сетей

**Rate Limiting:**

- Определение ошибки "Too Many Requests" (429)
- Exponential backoff для повторных попыток
- Постановка задачи в очередь на повторное выполнение
- Уведомление пользователя: "Задача приостановлена из-за лимита API. Повтор через X минут"

**Недействительный токен:**

- Определение ошибки "Invalid token" (401)
- Попытка обновления токена через refresh token
- При успехе - повтор запроса
- При неудаче - пометка аккаунта как неактивного, уведомление пользователя

**Недостаточно прав доступа:**

- Определение ошибки "Insufficient permissions" (403)
- Сообщение пользователю: "У вас нет прав для выполнения этого действия"
- Предложение проверить права доступа в настройках социальной сети

**Сервис недоступен:**

- Определение ошибок 500/503
- Использование circuit breaker pattern
- Retry логика (Polly)
- При неудаче - пометка задачи как Failed
- Сообщение: "Сервис временно недоступен. Попробуйте позже"

### 6.2 Обработка длительных задач

**Требования:**

- Сохранение промежуточных результатов в БД
- Периодическое обновление прогресса задачи
- Сохранение checkpoint'ов для длительных задач
- Возможность приостановки и возобновления задачи
- Уведомление пользователя о длительном выполнении

**Восстановление после сбоя:**

- Проверка активных задач при перезапуске
- Возобновление выполнения с последнего checkpoint
- При отсутствии checkpoint - пометка задачи как Failed

### 6.3 Обработка большого объема данных

**Требования:**

- Разбиение обработки на батчи (chunks)
- Обработка каждого батча отдельно
- Сохранение промежуточных результатов
- Обновление прогресса после каждого батча
- Продолжение обработки остальных батчей при ошибке одного
