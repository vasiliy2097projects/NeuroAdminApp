# NeuroAdmin Roadmap

## Обзор

Дорожная карта разработки NeuroAdmin - AI-powered SMM платформы с модулем очистки от ботов.

---

## 🎯 Цели проекта

1. Создать работающую систему аутентификации
2. Реализовать интеграции с VK и OK API
3. Разработать модуль очистки от ботов
4. Подготовить проект к production deployment

---

## 🔌 Release 2.0: Интеграции с социальными сетями

**Цель:** Реализовать сервисы для работы с API ВКонтакте и Одноклассников, а также возможность подключения соц сетей на Dashboard.

**Срок:** 3-4 недели

### Backend

- ⏳ VK Service (#9)

  - Интеграция с VK API
  - Получение профилей, подписчиков, друзей
  - OAuth авторизация
  - Управление токенами доступа
  - Rate limiting и кэширование

- ⏳ OK Service (#10)

  - Интеграция с OK API
  - OAuth авторизация
  - Управление токенами доступа
  - Аналогичный функционал VK Service

- ⏳ Social Accounts Management
  - Хранение подключенных аккаунтов
  - Управление OAuth токенами
  - API для получения списка подключенных аккаунтов

### Frontend

- ⏳ Dashboard с подключением соц сетей
  - Раздел подключения социальных сетей на Dashboard
  - OAuth flow для VK и Одноклассников
  - Список подключенных аккаунтов
  - Управление подключенными аккаунтами

### Design

- 🔄 Дизайн Dashboard (#19) - с разделом подключения соц сетей
- 🔄 Дизайн интеграции с социальными сетями (#21)

**Критерии готовности:**

- [ ] Пользователь может подключить VK аккаунт через OAuth
- [ ] Пользователь может подключить Одноклассники аккаунт через OAuth
- [ ] Подключенные аккаунты отображаются на Dashboard
- [ ] Можно получить данные пользователя из VK
- [ ] Можно получить данные пользователя из OK
- [ ] Rate limiting работает корректно
- [ ] Кэширование настроено

## 🤖 Release 3.0: Система задач и очистка от ботов

**Цель:** Реализовать систему задач, где пользователь может создавать различные задачи для подключенных социальных сетей. Первый тип задачи - очистка от ботов.

**Срок:** 6-8 недель

### Backend

- ⏳ Task Management Service (#26)

  - Создание задач различных типов
  - Управление задачами (статусы, прогресс)
  - История выполнения задач
  - API endpoints для работы с задачами

- ⏳ Bot Detection Service - синхронные операции (#11)

  - Создание задачи "Очистка от ботов"
  - Получение статуса и результатов задачи
  - API endpoints

- ⏳ Bot Detection Service - асинхронные операции (#12)
  - RabbitMQ интеграция
  - Background workers для обработки задач
  - Стратегии определения ботов
  - Интеграция с VK/OK Services

### Frontend

- ⏳ Task Management System (#24)

  - Главная страница задач
  - Форма создания задачи
  - Список задач с фильтрами
  - Страница деталей задачи
  - Страница результатов задачи

- ⏳ Frontend Bot Detection Module (#13)
  - Форма создания задачи "Очистка от ботов"
  - Страница результатов очистки от ботов
  - Визуализация данных
  - Экспорт результатов

### Design

- 🔄 Дизайн системы задач (#24)
- 🔄 Дизайн модуля очистки от ботов (#20)

**Критерии готовности:**

- [ ] Пользователь может создать задачу "Очистка от ботов" для подключенной соц сети
- [ ] Задача обрабатывается асинхронно
- [ ] Результаты отображаются в UI
- [ ] Стратегии определения ботов работают
- [ ] Можно экспортировать результаты
- [ ] Система задач готова для добавления новых типов задач (статистика, создание поста и т.д.)

## 🔄 Зависимости между задачами

```
Дизайн-система (#17)
    ↓
Дизайн страниц аутентификации (#18)
    ↓
Frontend Authentication (#8)

API Gateway (#5) → Auth Service (#6)
    ↓
Frontend Setup (#7)

VK Service (#9) ─┐
OK Service (#10) ─┼→ Social Accounts Management
                  └→ Dashboard (#19)
                        ↓
                  Task Management Service (#25)
                        ↓
                  Bot Detection Service (#11, #12)
                        ↓
                  Frontend Task Management (#24)
                        ↓
                  Frontend Bot Detection (#13)

Bot Detection Service (#11) → Bot Detection Service (#12)
    ↓
Frontend Bot Detection (#13)
```

## 📝 Заметки

- Все дизайн задачи должны быть завершены до начала разработки соответствующих frontend компонентов
- Backend сервисы могут разрабатываться параллельно
- Интеграционные тесты должны покрывать взаимодействие между сервисами
- Документация обновляется по мере разработки
